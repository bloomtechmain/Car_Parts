# CarParts Finder — Development Progress
**Date:** 18 June 2026

---

## Completed Tasks

### 1. Admin Notification Email on Customer Option Selection
- **Feature:** When a customer selects an option from the pricing email, the admin now receives an automatic email notification
- **Email contains:**
  - Customer name, email, and phone number
  - Car brand, model, and year
  - Ticket ID and ticket number
  - Part name and part category
  - Supplier name (whose quote was selected)
  - Selected option number, admin price (LKR), and delivery days
- **Files Changed:** `backend/src/services/emailService.ts` (new `sendAdminSelectionNotification` function), `backend/src/controllers/ticketController.ts`

---

### 2. Highlight Customer-Selected Option in Admin Dashboard
- **Feature:** When admin opens a closed ticket, the supplier reply that the customer selected is visually highlighted
- **Details:**
  - Selected reply card shows a green border (`border-green-500`) instead of the default border
  - A "Customer Selected This Option" banner with a green checkmark icon appears at the top of the card
  - `selected_reply_id` is saved to the `tickets` table when the customer confirms
- **Files Changed:** `frontend/src/components/dashboard/TicketDetailModal.tsx`, `frontend/src/types/index.ts`, `backend/src/controllers/ticketController.ts`, `backend/src/controllers/adminController.ts`

---

### 3. Phone Number Placeholder Changed to Sri Lanka Format
- **Issue:** The phone number field in the ticket creation form showed `+971 50 000 0000` (UAE format)
- **Fix:** Updated placeholder to `+94 77 000 0000` (Sri Lanka format)
- **Files Changed:** `frontend/src/components/find-parts/Step3ContactInfo.tsx`

---

### 4. Database Migrations — Missing Columns Fixed
- **Issue:** Production and local databases were missing columns required for the full quote flow, causing "Link Invalid" errors and server crashes on Railway
- **Missing columns added:**
  | Table | Column | Purpose |
  |-------|--------|---------|
  | `supplier_replies` | `delivery_days` | Supplier estimated delivery time |
  | `supplier_replies` | `admin_price` | Admin-set price shown to customer |
  | `tickets` | `options_token` | Secure token for customer selection email links |
  | `tickets` | `selected_reply_id` | Stores which option the customer selected |
- **Fix:** Added all 4 columns via migration scripts on local database and automatic startup migrations for production

---

### 5. Auto-Run Migrations on Server Startup
- **Feature:** Backend now automatically applies all database migrations every time the server starts
- **Details:**
  - Migrations use `IF NOT EXISTS` so they are safe to run multiple times
  - Server only starts listening after migrations complete successfully
  - If migrations fail, server exits with an error instead of running broken
  - This fixed the Railway production crash without needing manual database access
- **Files Changed:** `backend/src/server.ts`

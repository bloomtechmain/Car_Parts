## Issues Faced & How They Were Resolved

### 1. Git Push — Authentication Failed (HTTP 408 / 403)
**Problem:** First push timed out (408), second push gave permission denied (403).

**Cause:**
- 408 — Railway terminal can't open interactive browser auth
- 403 — Original token was a fine-grained PAT without write access to the repo

**Fix:** Replaced with a classic GitHub PAT (with `repo` scope). Token embedded directly in the remote URL:
```
https://NaduniTPerera:<token>@github.com/bloomtechmain/Car_Parts.git
```

---

### 2. Railway Build Error — Directory Not Found
**Problem:**
```
error creating app: directory Car Parts eCommerce/backend does not exist
```

**Cause:** The git repo was initialized inside `Car Parts eCommerce/`, so the repo root IS that folder. Setting Root Directory to `Car Parts eCommerce/backend` looked one level too deep.

**Fix:** Left Root Directory empty (repo root). Created `railway.toml` at repo root to handle both frontend and backend builds.

---

### 3. SQL Migration — `syntax error at or near "LIMIT"`
**Problem:** Pasting the full migration SQL into Railway's Postgres query runner kept throwing a LIMIT syntax error.

**Cause:** Railway's built-in query runner automatically appends `LIMIT` to every `SELECT` statement it detects — including the `SELECT INTO` inside the PL/pgSQL function body, which broke the `$$` block.

**Fix:**
- Rewrote the ticket number function to use a PostgreSQL `SEQUENCE` + `nextval()` instead of `SELECT COUNT(*)`
- No `SELECT` inside the function body → Railway can't inject LIMIT
- Split into **two separate queries** (tables/indexes/admin first, function second)

---

### 4. Login Error — CSP Violation / Calling localhost in Production
**Problem:**
```
Connecting to 'http://localhost:3001/api/auth/login' violates Content Security Policy
```

**Cause:** `VITE_API_URL` was not set on Railway (intentionally), but the fallback in `api.ts` was:
```typescript
baseURL: import.meta.env.VITE_API_URL ?? 'http://localhost:3001'
```
Since `VITE_API_URL` was `undefined`, it fell back to `localhost:3001` — a different origin — which Helmet's CSP (`default-src 'self'`) blocked.

**Fix:** Changed fallback to empty string:
```typescript
baseURL: import.meta.env.VITE_API_URL ?? ''
```
Empty string = same origin. Frontend calls `/api/auth/login` on the Railway domain. No CORS, no CSP violation.
